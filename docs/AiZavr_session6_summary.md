## Что произошло: смена модели сжатия

### Почему старая модель (D-051..D-055) не годилась
Старая модель: единица сжатия — структурный отрезок между смежными развилками,
якорь на каждый отрезок, крепление к началу отрезка.

Вскрылась структурная дыра: сжатие середины ветки — это lossy-коммит, меняющий
контекст для всего, что ниже. Если сжать отрезок А-Б, то вопрос В ниже может
опираться на деталь, которую сжатие выбросило → В «повисает в пустоте».
А если ниже развилка (В1/В2/В3) — одним сжатием их не обслужить, они опираются
на разные части Б. Вывод: **сжать середину, сохранив несколько разных потомков,
нельзя без пересжатия каждого потомка.**

Корень ошибки: развилка — событие *топологическое*, а «где кончилась вода» —
граница *семантическая*. Они не совпадают. Старая модель уложила сжатие в
топологию дерева ценой подмены смысла.

### Новая модель (D-057..D-068)
- **Единица сжатия — произвольный линейный диапазон**, заданный пользователем,
  а не отрезок между развилками (D-057).
- **Две границы-маркера на одной линии:** верх (откуда) — A-узел/маркер,
  низ (докуда) — конкретный узел. Путь линеен по определению, боковые ветви
  не входят и не трогаются. Низ не обязан быть маркером — может быть узлом
  в фокусе («удобный» сценарий), без авто-доведения до листа (D-059).
- **Результат S** (узел compressed_summary) крепится **веткой-сестрой** к
  верхнему A в Q-слот, рядом со старым продолжением. Исходная линия цела (D-060).
- **Под S — служебная заглушка** node_type=`compression_placeholder` в A-слоте,
  БЕЗ вызова LLM, сохраняет чередование Q→A, в модель не отправляется. Текст
  опционален (плагин может вернуть пустую). Тип назначает ядро, текст — плагин (D-061).
- **При следующем запросе** в LLM уходит S+новый вопрос; заглушка не отправляется.
- **Уровни расщеплены на две оси:** диапазон (пользователь, маркеры) и сила
  (плагин). L0 Hidden — чистая UI-свёртка без сжатия/вызова/узла. L1/L2/L3 —
  пресеты силы. «Резюме всей беседы» = частный случай диапазона корень→лист (D-062).
  D-003 (4 дискретные кнопки) сохранён.
- **Якорь как «первая Q+A пара отрезка» отменён** вместе с отрезками; дословно
  остаётся всё выше верхнего маркера (D-063).
- **Несколько диапазонов** сжимаются независимо, несколько пар маркеров (D-064).

## Маркеры как примитив (D-058)
- `(id, node_id, label, comment?)`, persistent, только на A-узлах.
- CRUD вручную пользователем; ядро не создаёт маркеры автоматически.
- Переиспользуемый примитив: сейчас потребитель один (сжатие), позже —
  навигация по графу, экспорт.
- **Почему только A:** ответ — единственная точка на линии (один родитель).
  Вопрос неоднозначен (несколько веток на один ответ).
- **Целостность (D-067):** маркер нельзя удалить, пока на его node_id ссылается
  объект (узел compressed_summary в extra.compression). Признак «используется»
  ВЫЧИСЛЯЕТСЯ поиском ссылок, НЕ хранится флагом (флаг рассинхронизируется —
  та же болезнь, что денормализованный children_count). Учёт ведёт ядро.
  Реестр обратных ссылок — отложенная оптимизация, не сейчас.

## Граница ядро/плагин (D-066, усиливает D-056)
**Ядро (топология + хранение, детерминировано):**
- `list_startable_markers` — маркеры, годные как начало (не-лист)
- `list_reachable_markers(from)` — достижимые концы по линии (маркеры + листья)
- `resolve_linear_range(a, b)` — путь + проверка «на одной линии» (или ошибка)
- `attach_compressed(anchor, S_text, placeholder_text?, plugin_meta)` — крепление
- CRUD маркеров

**Плагин (политика, субъективно):**
- выбор пары маркеров, двухшаговый UX-сценарий выбора (D-068)
- алгоритм сжатия, промпт, уровни/сила
- тексты S и заглушки
- самоописание: plugin_id, plugin_version, algorithm, params

Плагин НЕ обходит дерево сам — получает результаты топологических запросов ядра.
Это защищает плагины от миграций схемы.

## Метрика происхождения (D-065)
В `extra` узла S: `compression = { start_node_id, end_node_id, created_at` (ядро)`;
plugin_id, plugin_version, algorithm, params` (плагин, непрозрачно для ядра)` }`.
Отдельной таблицы для сжатий нет. created_at нужен в т.ч. для разрешения
коллизий нескольких сжатий одного диапазона (ср. Q-005).
