### Frontend (App.tsx, App.css)
- Режим развилки: горизонтальный ряд карточек между сообщениями
  - выбор мышью (клик) или Ctrl+Left/Right + Enter/Ctrl+Down

  - "+" (синяя) — создать новую ветку

  - Ctrl+Down — "провалиться" в выбранную ветку (подтвердить выбор, D-045)
  - Ctrl+Left/Right — перебор карточек в режиме развилки
- Переименование карточек:
  - двойной клик или F2 — режим редактирования имени

## Решения и принципы, закреплённые в сессии
- Визуализация дерева диалога (навигатор) — ПЛАГИН, не ядро.
  Ядро предоставляет: сплит-панель (resizable/collapsible), API точки
  входа-выхода (дерево, навигация, события). Сам навигатор — отдельная
  задача, реализуется позже. Соответствует D-041 (plugin-friendly).
- Пузыри сообщений: 60-70% ширины текстового фрейма, лево-право сохраняется.
- Денормализация children_count оправдана: узлы не удаляются/не редактируются
  в текущей фазе, счётчик инкрементируется только в create_node
- Атомарные backend-операции вместо многошаговых вызовов с фронта
  (урок из бага select_branch: частичный сбой при цепочке вызовов)
- Дизайн отлаживаем ПОТОМ, разом; сейчас приоритет — структура
- Автогенерация имён веток через LLM ОТЛОЖЕНА до слоя ролей (v0.2),
  чтобы не плодить техдолг; сейчас branch_name = первые 128 символов вопроса
- Ручное переименование сделано сейчас (не переписываемый фундамент)
